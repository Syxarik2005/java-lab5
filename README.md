# Лабораторная работа №5: Рефлексия и Внедрение Зависимостей

## Описание проекта

Данный проект представляет собой реализацию собственного мини-фреймворка для **Внедрения Зависимостей (Dependency Injection, DI)** на языке Java.

Основная задача — создать класс `Injector`, который, используя **Java Reflection API**, способен автоматически инициализировать поля в любом объекте. Эти поля должны быть помечены специальной аннотацией `@AutoInjectable`. Конкретные классы для инициализации определяются во внешнем файле конфигурации (`.properties`), что позволяет менять поведение программы без перекомпиляции кода.

## Ключевые концепции

*   **Рефлексия (Reflection):** Механизм, позволяющий программе "исследовать" саму себя во время выполнения. `Injector` использует рефлексию, чтобы находить приватные поля, проверять их аннотации и изменять их значения.
*   **Внедрение Зависимостей (DI):** Принцип, при котором зависимости (объекты, необходимые классу для работы) не создаются внутри самого класса, а "внедряются" извне. Это делает код более гибким, модульным и легко тестируемым.

## Как это работает

1.  Создается кастомная аннотация `@AutoInjectable`.
2.  Поля в классе (например, `SomeBean`), которые нуждаются во внедрении, помечаются этой аннотацией. Типом поля должен быть интерфейс.
3.  Создается файл `config.properties`, в котором указывается, какой конкретный класс должен быть создан для каждого интерфейса (например, `com.sergey.SomeInterface=com.sergey.SomeImpl`).
4.  Класс `Injector` читает этот файл конфигурации.
5.  При вызове метода `injector.inject(object)`, инжектор сканирует все поля объекта, находит помеченные `@AutoInjectable`, создает экземпляры нужных классов и "внедряет" их в эти поля.

## Технологии

*   Java 24 (Amazon Corretto)
*   Maven - для управления зависимостями и сборкой
*   Java Reflection API
*   JUnit 5 - для модульного тестирования

## Как собрать и запустить

1.  **Клонируйте репозиторий.**
2.  **Запуск тестов:**
    Эта команда запустит все Unit-тесты, которые доказывают корректность работы инжектора в различных ситуациях.
    ```bash
    mvn test
    ```
3.  **Запуск демонстрации:**
    Эта команда запустит основной класс `Main`, который продемонстрирует успешное внедрение зависимостей.
    ```bash
    mvn compile exec:java -Dexec.mainClass="com.sergey.Main"
    ```
4.  **Генерация документации:**
    ```bash
    mvn javadoc:javadoc
    ```
